---

Dexer gives .NET developers a complete object model for working with Android Dalvik Executable (`.DEX`) files — the binary format executed by the Android Runtime. You can read existing DEX files, inspect and modify classes, methods, fields, and individual bytecode instructions, then write the result back to disk.

## Get started

<Steps>
  <Step title="Install Dexer">
    Add the NuGet package to your project:

    ```bash
    dotnet add package Dexer
    ```
  </Step>
  <Step title="Load a DEX file">
    Read a `.DEX` file into the object model with a single call:

    ```csharp
    using Dexer.Core;

    Dex dex = Dex.Read("classes.dex");
    ```
  </Step>
  <Step title="Inspect and modify">
    Navigate the class hierarchy and modify instructions:

    ```csharp
    using Dexer.Instructions;

    MethodDefinition method = dex.GetClass("com.example.MainActivity")
                                  .GetMethod("onCreate");

    // Change an opcode
    method.Body.Instructions[5].OpCode = OpCodes.AddInt;

    // Change a string constant
    method.Body.Instructions[7].Operand = "Hello from Dexer!";
    ```
  </Step>
  <Step title="Save the result">
    Write the modified DEX back to disk:

    ```csharp
    dex.Write("output.dex");
    ```
  </Step>
</Steps>

<Note>
  Dexer targets **.NET Standard 2.0**, making it compatible with .NET 6+, .NET Framework 4.6.1+, and any other .NET Standard 2.0-compliant runtime.
</Note>

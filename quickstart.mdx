This guide walks you through the complete Dexer workflow: install the package, load a `.DEX` file, find a class and method, change an opcode and a string constant, then save the patched file. You will have working code in about five minutes.

<Note>
  You need a `.DEX` file to follow along. If you do not have one, extract `classes.dex` from any Android APK by renaming the `.apk` to `.zip` and unpacking it.
</Note>

<Steps>

<Step title="Install the Dexer package">

Add Dexer to your .NET project using the dotnet CLI or by adding a `PackageReference` directly to your project file.

<CodeGroup>

```bash dotnet CLI
dotnet add package Dexer
```

```xml PackageReference
<PackageReference Include="Dexer" Version="1.0.0" />
```

</CodeGroup>

Dexer targets .NET Standard 2.0 and works with .NET 6+, .NET Framework 4.6.1+, and any compatible runtime.

</Step>

<Step title="Add using directives">

At the top of your source file, import the three namespaces you will use most often.

```csharp
using Dexer.Core;
using Dexer.Instructions;
```

`Dexer.Core` contains `Dex`, `ClassDefinition`, `MethodDefinition`, `Prototype`, and related types. `Dexer.Instructions` contains `Instruction`, `OpCodes`, and `Register`.

</Step>

<Step title="Load a DEX file">

Call `Dex.Read` with a file path. This parses the entire DEX binary into an in-memory object model.

```csharp
Dex dex = Dex.Read("classes.dex");
```

You can also read from a `Stream` if you are working with files inside an APK or a custom source:

```csharp
using Stream stream = File.OpenRead("classes.dex");
Dex dex = Dex.Read(stream);
```

</Step>

<Step title="Navigate to a class and method">

Use `GetClass` on the `Dex` object to look up a class by its fully-qualified name, then call `GetMethod` on the returned `ClassDefinition`.

```csharp
ClassDefinition cls = dex.GetClass("dexer.poc.MainActivity");
MethodDefinition method = cls.GetMethod("onCreate");
```

<Tip>
  `GetClass` searches inner classes recursively. `GetMethod` returns the first match by name. Use `GetMethods` (plural) to retrieve all overloads when a class has multiple methods with the same name.
</Tip>

Both methods return `null` if there is no match, so check for `null` when working with unknown DEX files:

```csharp
ClassDefinition? cls = dex.GetClass("dexer.poc.MainActivity");
MethodDefinition? method = cls?.GetMethod("onCreate");
```

</Step>

<Step title="Inspect and modify instructions">

Each `MethodDefinition` with a body exposes `method.Body.Instructions`, a list of `Instruction` objects. Each `Instruction` has an `OpCode` property and an `Operand` property.

The example below replicates the canonical Dexer demo: it changes a multiply opcode to add, and replaces a string constant.

```csharp
// Change the arithmetic opcode at index 5 from multiply to add
method.Body.Instructions[5].OpCode = OpCodes.AddInt;

// Replace the string constant at index 7
method.Body.Instructions[7].Operand = "Dexer rocks! ";
```

You can also create and insert entirely new instructions. The `Instruction` constructor accepts an `OpCodes` value, an optional operand, and a `params` array of registers:

```csharp
// Load an integer constant into register 1
var registers = method.Body.Registers;
Instruction iconst = new Instruction(OpCodes.Const, unchecked((int)0xFFFF00FF), registers[1]);
method.Body.Instructions.Insert(14, iconst);

// Invoke a method with 'this' in register 5 and the color in register 1
Prototype prototype = new Prototype(PrimitiveType.Void, new Parameter(PrimitiveType.Int));
MethodReference setTitleColor = dex.Import(
    new MethodReference(method.Owner, "setTitleColor", prototype));

Instruction iinvoke = new Instruction(OpCodes.InvokeVirtual, setTitleColor, registers[5], registers[1]);
method.Body.Instructions.Insert(15, iinvoke);
```

<Warning>
  Instruction indices are position-based. After inserting instructions, all subsequent indices shift. Insert from the end of the method body backwards, or track instructions by reference rather than index, to avoid off-by-one errors.
</Warning>

</Step>

<Step title="Write the patched DEX file">

Call `Write` with an output file path. Dexer recalculates the file header, string pool, and checksums automatically.

```csharp
dex.Write("output.dex");
```

You can also write to a `Stream`:

```csharp
using Stream output = File.Create("output.dex");
dex.Write(output);
```

</Step>

</Steps>

## Complete example

Here is the full program from the steps above, matching the canonical Dexer README example:

```csharp
using Dexer.Core;
using Dexer.Instructions;

Dex dex = Dex.Read("classes.dex");
MethodDefinition method = dex.GetClass("dexer.poc.MainActivity").GetMethod("onCreate");

// Patch opcode: change multiply to add
method.Body.Instructions[5].OpCode = OpCodes.AddInt;

// Patch string constant
method.Body.Instructions[7].Operand = "Dexer rocks! ";

dex.Write("output.dex");
```

## Rebuild and reinstall the APK

After writing the patched DEX, you can rebuild and reinstall the APK using [apktool](https://apktool.org/) and the Android SDK tools:

```bash
# Unpack the APK (keep raw classes.dex)
apktool d -s -f MyApp.apk output/

# Copy your patched DEX into place
cp output.dex output/classes.dex

# Rebuild the APK
apktool b output/ MyApp.patched.apk

# Sign with the debug key
jarsigner -keystore ~/.android/debug.keystore \
          -storepass android -keypass android \
          MyApp.patched.apk androiddebugkey

# Install on a connected device
adb install MyApp.patched.apk
```

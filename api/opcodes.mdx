title: "OpCodes — Dalvik instruction set reference"
description: "Complete enumeration of Dalvik OpCodes values used by Dexer's Instruction class, organized by category with hex values and encoding notes."

`OpCodes` is a C# `enum` in the `Dexer.Instructions` namespace. Every member maps directly to a Dalvik bytecode value as specified in the Android DEX format. You assign a member to `Instruction.OpCode` to set what an instruction does; Dexer uses the value to determine the instruction's encoding size and operand format during serialization.

The `Nop` opcode (`0x00`) is also used as the padding byte between code sections. The pseudo-opcode table (`PseudoOpCodes`) contains three additional values — `PackedSwitch` (`0x0100`), `SparseSwitch` (`0x0200`), and `FillArrayData` (`0x0300`) — that represent out-of-line data blocks written after the main instruction stream; you do not set these directly on an `Instruction`.

<AccordionGroup>

  <Accordion title="Move operations">
    Move instructions copy register values between registers. The suffix indicates the register width or addressing mode: `Wide` operates on 64-bit (long/double) register pairs, `Object` preserves the GC reference type, `From16` / `16` allow larger register indices.

    | Name               | Hex    | Encoding        |
    |--------------------|--------|-----------------|
    | `Move`             | `0x01` | vA, vB          |
    | `MoveFrom16`       | `0x02` | vAA, vBBBB      |
    | `Move16`           | `0x03` | vAAAA, vBBBB    |
    | `MoveWide`         | `0x04` | vA, vB          |
    | `MoveWideFrom16`   | `0x05` | vAA, vBBBB      |
    | `MoveWide16`       | `0x06` | vAAAA, vBBBB    |
    | `MoveObject`       | `0x07` | vA, vB          |
    | `MoveObjectFrom16` | `0x08` | vAA, vBBBB      |
    | `MoveObject16`     | `0x09` | vAAAA, vBBBB    |
    | `MoveResult`       | `0x0a` | vAA             |
    | `MoveResultWide`   | `0x0b` | vAA             |
    | `MoveResultObject` | `0x0c` | vAA             |
    | `MoveException`    | `0x0d` | vAA             |

    `MoveResult*` must be the instruction immediately following an `invoke-*` or `filled-new-array` instruction. `MoveException` must be the first instruction of an exception handler block.
  </Accordion>

  <Accordion title="Return operations">
    Return instructions exit the current method and optionally pass a value back to the caller.

    | Name           | Hex    | Encoding |
    |----------------|--------|----------|
    | `ReturnVoid`   | `0x0e` | (none)   |
    | `Return`       | `0x0f` | vAA      |
    | `ReturnWide`   | `0x10` | vAA      |
    | `ReturnObject` | `0x11` | vAA      |

    Use `ReturnVoid` for `void` methods. `ReturnWide` is for `long` or `double` return types.
  </Accordion>

  <Accordion title="Constant loading">
    Const instructions load an immediate literal value into a register. The `Instruction.Operand` must be the literal value (`int` or `long`), except for string and class constants which use `string` and `TypeReference` respectively.

    | Name                | Hex    | Encoding             | Notes                                 |
    |---------------------|--------|----------------------|---------------------------------------|
    | `Const4`            | `0x12` | vA, #+B              | 4-bit signed literal                  |
    | `Const16`           | `0x13` | vAA, #+BBBB          | 16-bit signed literal                 |
    | `Const`             | `0x14` | vAA, #+BBBBBBBB      | 32-bit literal                        |
    | `ConstHigh16`       | `0x15` | vAA, #+BBBB0000      | High 16 bits; low 16 set to zero      |
    | `ConstWide16`       | `0x16` | vAA, #+BBBB          | 16-bit sign-extended to 64 bits       |
    | `ConstWide32`       | `0x17` | vAA, #+BBBBBBBB      | 32-bit sign-extended to 64 bits       |
    | `ConstWide`         | `0x18` | vAA, #+BBBBBBBBBBBBBBBB | 64-bit literal                     |
    | `ConstWideHigh16`   | `0x19` | vAA, #+BBBB000000000000 | High 16 bits; low 48 set to zero  |
    | `ConstString`       | `0x1a` | vAA, string@BBBB     | Operand: `string`                     |
    | `ConstStringJumbo`  | `0x1b` | vAA, string@BBBBBBBB | Operand: `string`, large string pool  |
    | `ConstClass`        | `0x1c` | vAA, type@BBBB       | Operand: `TypeReference`              |
  </Accordion>

  </Accordion>

  <Accordion title="Type operations">
    These instructions check, cast, and create object instances and arrays.

    | Name                  | Hex    | Encoding                  | Operand           |
    |-----------------------|--------|---------------------------|-------------------|
    | `CheckCast`           | `0x1f` | vAA, type@BBBB            | `TypeReference`   |
    | `InstanceOf`          | `0x20` | vA, vB, type@CCCC         | `TypeReference`   |
    | `ArrayLength`         | `0x21` | vA, vB                    | (none)            |
    | `NewInstance`         | `0x22` | vAA, type@BBBB            | `TypeReference`   |
    | `NewArray`            | `0x23` | vA, vB, type@CCCC         | `TypeReference`   |
    | `FilledNewArray`      | `0x24` | \{vD,vE,vF,vG,vA\}, type@CCCC | `TypeReference` |
    | `FilledNewArrayRange` | `0x25` | \{vCCCC..vNNNN\}, type@BBBB | `TypeReference`   |

  </Accordion>

  <Accordion title="Method invocation">
    Invoke instructions call a method referenced by a `MethodReference` operand. The standard forms accept up to five explicit registers; the `*Range` forms accept a contiguous range of registers for methods with more arguments.

    **Standard forms** — encoding: \{vD,vE,vF,vG,vA\}, meth@CCCC

    | Name               | Hex    | Dispatch              |
    |--------------------|--------|-----------------------|
    | `InvokeVirtual`    | `0x6e` | virtual dispatch       |
    | `InvokeSuper`      | `0x6f` | super-class dispatch   |
    | `InvokeDirect`     | `0x70` | direct / constructor   |
    | `InvokeStatic`     | `0x71` | static method          |
    | `InvokeInterface`  | `0x72` | interface dispatch     |

    **Range forms** — encoding: \{vCCCC..vNNNN\}, meth@BBBB

    | Name                    | Hex    |
    |-------------------------|--------|
    | `InvokeVirtualRange`    | `0x74` |
    | `InvokeSuperRange`      | `0x75` |
    | `InvokeDirectRange`     | `0x76` |
    | `InvokeStaticRange`     | `0x77` |
    | `InvokeInterfaceRange`  | `0x78` |

    Follow any non-void invocation with a `MoveResult`, `MoveResultWide`, or `MoveResultObject` instruction to capture the return value.
  </Accordion>

  <Accordion title="Unary operations and type conversions">

    | `IntToDouble`    | `0x83` | int → double        |
    | `LongToInt`      | `0x84` | long → int          |
    | `LongToFloat`    | `0x85` | long → float        |

## Usage example

```csharp
using Dexer.Instructions;

// Load integer literal 42 into v0
var load = new Instruction(OpCodes.Const16, 42, body.Registers[0]);

// Add v0 + v1, store in v2
var add = new Instruction(
    OpCodes.AddInt,
    null,
    body.Registers[2],  // destination
    body.Registers[0],  // source A
    body.Registers[1]   // source B
);

// Return the int in v2
var ret = new Instruction(OpCodes.Return, null, body.Registers[2]);

body.Instructions.Add(load);
body.Instructions.Add(add);
body.Instructions.Add(ret);
```

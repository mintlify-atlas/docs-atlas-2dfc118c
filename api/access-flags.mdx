`AccessFlags` encodes the access modifiers and behavioral attributes of DEX declarations. It is applied to `ClassDefinition`, `MethodDefinition`, and `FieldDefinition` objects through their `AccessFlags` property. Because the enum is decorated with `[Flags]` and uses `uint` as its underlying type, multiple flags are combined with bitwise OR and tested with bitwise AND.

```csharp
[Flags]
public enum AccessFlags : uint
```

The class lives in the `Dexer.Core` namespace.

---

## Flag values

### Flags applicable to classes

| Flag | Hex value | Description |
|---|---|---|
| `Public` | `0x1` | The class is visible to all callers. |
| `Private` | `0x2` | The class is visible only within its defining class. Inner classes only. |
| `Protected` | `0x4` | The class is visible to subclasses of its enclosing class. Inner classes only. |
| `Static` | `0x8` | The class does not hold an implicit reference to an enclosing instance. Inner classes only. |
| `Final` | `0x10` | The class cannot be subclassed. |
| `Interface` | `0x200` | The declaration is an interface, not a concrete class. |
| `Abstract` | `0x400` | The class may contain abstract methods and cannot be instantiated directly. |
| `Synthetic` | `0x1000` | The class was generated by the compiler and does not appear in the source code. |
| `Annotation` | `0x2000` | The declaration is an annotation type (`@interface`). |
| `Enum` | `0x4000` | The declaration is an enumeration type. |

### Flags applicable to methods

| Flag | Hex value | Description |
|---|---|---|
| `Public` | `0x1` | The method is callable by any class. |
| `Private` | `0x2` | The method is callable only within the defining class. |
| `Protected` | `0x4` | The method is callable within the package and by subclasses. |
| `Static` | `0x8` | The method does not take an implicit `this` parameter. |
| `Final` | `0x10` | The method cannot be overridden by a subclass. |
| `Synchronized` | `0x20` | The method acquires the monitor of `this` (or the class) before executing. |
| `Bridge` | `0x40` | The method is a bridge method generated by the compiler (e.g., for covariant return types). |
| `VarArgs` | `0x80` | The last parameter is a variable-length argument list. |
| `Native` | `0x100` | The method body is implemented in native code (JNI). |
| `Abstract` | `0x400` | The method has no body; must be overridden by a concrete subclass. |
| `StrictFp` | `0x800` | Floating-point operations follow strict IEEE 754 semantics. |
| `Synthetic` | `0x1000` | The method was generated by the compiler and does not appear in source. |
| `Constructor` | `0x10000` | The method is an instance initializer (`<init>`) or static initializer (`<clinit>`). |
| `DeclaredSynchronized` | `0x20000` | The method is declared `synchronized` in the source but is stored differently in the DEX format. |

### Flags applicable to fields

| Flag | Hex value | Description |
|---|---|---|
| `Public` | `0x1` | The field is accessible by any class. |
| `Private` | `0x2` | The field is accessible only within the defining class. |
| `Protected` | `0x4` | The field is accessible within the package and by subclasses. |
| `Static` | `0x8` | The field belongs to the class, not to individual instances. |
| `Final` | `0x10` | The field cannot be reassigned after initial assignment. |
| `Volatile` | `0x40` | Reads and writes to this field are not cached; they go directly to main memory. |
| `Transient` | `0x80` | The field is excluded from serialization. |
| `Synthetic` | `0x1000` | The field was generated by the compiler and does not appear in source. |
| `Enum` | `0x4000` | The field holds an enum constant value. |

---

## Overlapping values

Two pairs of flags share the same numeric value. The DEX specification uses the same bit position for different purposes depending on the declaration context:

<Warning>
  `Bridge` and `Volatile` both equal `0x40`. `VarArgs` and `Transient` both equal `0x80`. When reading an `AccessFlags` value, the declaration type — method or field — determines which name is semantically correct. Dexer's C# enum simply exposes both names as aliases for the same bit.
</Warning>

| Bit | When applied to a method | When applied to a field |
|---|---|---|
| `0x40` | `Bridge` | `Volatile` |
| `0x80` | `VarArgs` | `Transient` |

There is also one reserved value:

| Flag | Hex value | Description |
|---|---|---|
| `Unused` | `0x8000` | Reserved; not defined by the Dalvik specification. |

```csharp

public enum AccessFlags : uint

    Public              = 0x1,

    Static              = 0x8,
    Final               = 0x10,
    Synchronized        = 0x20,
    Volatile            = 0x40,
```

## Usage examples

### Testing individual flags

Use bitwise AND to test whether a specific flag is set:

```csharp
// Check if a class is public and abstract
bool isPublicAbstract = (classdef.AccessFlags & AccessFlags.Public)   != 0
                     && (classdef.AccessFlags & AccessFlags.Abstract) != 0;
```

### Combining flags

Use bitwise OR to construct an `AccessFlags` value:

```csharp
// Mark a method as public and static
methodDef.AccessFlags = AccessFlags.Public | AccessFlags.Static;

// Mark an interface as public and abstract (typical for interface class definitions)
classDef.AccessFlags = AccessFlags.Public | AccessFlags.Interface | AccessFlags.Abstract;
```

### Checking for enum types and annotation types

```csharp
bool isEnum       = (classDef.AccessFlags & AccessFlags.Enum)       != 0;
bool isAnnotation = (classDef.AccessFlags & AccessFlags.Annotation) != 0;
```

### Distinguishing constructors from regular methods

```csharp
foreach (var method in classDef.Methods)
{
    bool isConstructor = (method.AccessFlags & AccessFlags.Constructor) != 0;
    Console.WriteLine($"{method.Name}: constructor={isConstructor}");
}
```

### Context-sensitive flag interpretation

When interpreting flags on a `FieldDefinition`, treat `0x40` as `Volatile` and `0x80` as `Transient`. When interpreting flags on a `MethodDefinition`, treat the same bits as `Bridge` and `VarArgs` respectively:

```csharp
// Field context
bool isVolatile  = (fieldDef.AccessFlags & AccessFlags.Volatile)  != 0;
bool isTransient = (fieldDef.AccessFlags & AccessFlags.Transient) != 0;

// Method context — same bit positions, different names
bool isBridge  = (methodDef.AccessFlags & AccessFlags.Bridge)  != 0;
bool isVarArgs = (methodDef.AccessFlags & AccessFlags.VarArgs) != 0;
```
